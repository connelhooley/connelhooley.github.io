Introduction

Don’t need to be a Dev to use it. Tagreted at business users who know a bit about data. Data sources -> dataset/model -> reports -> dashboard Need to upload a PBIX file. This is called a PowerBI desktop document. It contains a ready made model/dataset. Can also upload spreadsheets and CSVs. If you need to combine multiple sources you have to create a dataset that combines them. If not you can just upload the singular file.

Data Sources

Used to mash data from various sources

Load Data

Databases, files and web services.

Import

Copies data into Power BI dataset in the cloud. Data needs to be updated when it changes.

DirectQuery

Power BI queries data on prem when a report is viewed. Requires Enterprise Gatway (?) for on prem. Does not allow you to mash multiple data sources, all data must come from a single database. No calculated columns or tables No data type changes Azure Datawarehouse does not require enterprise gateway

Manual

You can also manually enter data via a grid GUI inside Power BI.

Model

Contains meta data like the relationships and formatting of data. Contains multiple a queries.

Queries

A query is a table or file/source. Queries can be grouped. Can make top row headers. Can remove rows. Power BI desktop tries to figure out data types in columns, you can set it manually. Can filter rows by column, numeric or string based comparison. Can chose columns to keep or columns to remove. Can add data from one query into another by using joins. Can split a column by a delimeter or number of characters. Queries can be shared around an organisation for people to create their models from.

Model

Queries become tables in the model. Can link tables together using relationships. Power BI desktop attempts to create relationships for you. Relationships must be on a single column on two tables.

Computed columns are made using DAX: Data Analysis eXpressions. Similar to Excel formulas. Computed columns take up additonal memory in model. Computed values can be inside queries or models.

Numerical columns are configured to be summerized by default. You can turn this off per column. It prevents relationships being set up on the column. It enables visuals that do counts, averages etc. on the column.

Can use the related function to refer to values in another releated table Can use hierachies to allow drill downs. Can set another column to be the sort column, then hide the column that is used - useful for string values. Can give a column a category. E.g. tell Power BI that it is a city or postcode, required for some visuals I think.

Measures are an aggregation against a table. E.g. total unique rows.

Report

Reports are a collection of visuals. They can be created in PowerBI online or inside the PBIX or Excel file. Reports can have multiple pages (like spreadsheet tabs). All pages must share the same single dataset.

Visuals

Visuals are graphical representations of data. Select columns from tables in model. Power BI then uses the visual it thinks is best. You can then change it. They can be tables or charts or cards. PowerBI autogenerates some visuals for you called quick insights. Can get developers to write custom visuals.

Card is a single value, can be used with measures.

Filters

Filters can filter entire report, entire page or a single visual. Slicer is a filter visual for the page(?).

Highlighting

Highlighting is similar to filtering. It highlights the matching data, but does not hide unmatching. Visuals that do not support highlighting just filter.

Appearance

Can adjust properties on a visual to change how it looks. Power BI desktop gives you more options here. Can also add shapes to report.

Dashboards

Dashboards are groups of tiles. A tile can be an entire report, or a single visual. You can also create tiles that point to a wide variety of media such as a web page, video or image. The reports and visuals inside a dashboard do not have to come from the same dataset. Can “pin live page” this means any changes to report are also shown in dashboard. Can also ask natural language questions inside dashboard.

Publishing

You publish a PBIX file. This uploads model, data and reports to the cloud.

Can upload a file on your machine or you can upload from one drive.

If using OneDrive you can configure Power BI Online to automatically reset model, data and reports when the file changes. Requires Power BI Pro.

Excel files do not require a gateway for auto-refresh.

Power BI Online

Cannot change queries or model. Can change reports.

Gateway is required for any form of on prem database. It is a bridge into the local network from Power BI online. ~ once an hour

Manual entry requires model to be re-uploaded.
